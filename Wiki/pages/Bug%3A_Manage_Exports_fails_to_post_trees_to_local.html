Bug: Manage Exports fails to post trees to local

From Dragon Age Toolset Wiki
Jump to: navigation, search
  • Version found: 1.0.1088.0
  • Status: Open


 This issue has been recorded in BioWare's internal tracking system.

Reference number: 216817 -BryanDerksen

Description

When I run Post Trees To Local via Manage Exports it is reporting errors and failing however when I run Post Trees To Local via the toolbar button it does not. For example, the following is the output from running Post Trees To Local via Manage Exports: 

S: 12:22:37 - Export Summary, 1 Requested, 0 Success, 0 Skipped, 1 Failed.  Please see below for more details.
E: 12:22:37 - mbtf01.lvl - Exporter FAILED for resource: mbtf01.lvl
E: 12:22:37 - Failed to export trees for layout [mbtf01d]
E: 12:22:37 - Failed to create composite Speed Tree resources
E: 12:22:37 - Speed Tree Resource Creation Failed: Could not validate texture C:\Users\LAUREN~1\AppData\Local\Temp\SpeedTreePackage\Output\tre_f_confsapa_Diffuse_0.tga
E: 12:22:37 - SpeedTreeCAD output texture C:\Users\LAUREN~1\AppData\Local\Temp\SpeedTreePackage\Output\tre_f_confsapa_Diffuse_0.tga had an invalid size of 161, 256
S: 12:22:25 - Starting export: "Post Trees to Local"

Whereas the following is from running Post Trees To Local via the toolbar button: 

S: 12:40:09 - Export Summary, 1 Requested, 1 Success, 0 Skipped, 0 Failed.  Please see below for more details.
S: 12:22:37 - Starting export: "Post Selection to Local"

Workarounds

Use the Post Trees To Local toolbar button.
